# Hackaton_Avito_Podrabotka_SF_MIPHI
Рабочий репозиторий для команды 14 прохождения кейса №2 от Авито-Подработка по бинарной клссификации работников по готовности выйти на смену.

Финальный вариант работы в файле Hackaton_Avito_14.ipynb, который содержит функции: 
- для загрузки и валидации данных,
- разбиения датасета на обучающую и тестовую выборки,
- построения признаков, включая целевые,
- запуска пайплайна с выбронной моделью
- обучения модели
- подсчет метрики
  
Разбиение датасета на train и test сделано по start_at ак квантиль 0.8 в трейн и 0.2 в тест по фиксированной дате и учитывает недельные циклы, что помогает избегать утечки данных.

**Построение признаков**
Устраняется утечка данных после shift start и все исторические признаки строятся только на выбранном датасете до момента начала взаимодействия.
Помимо объединения построены признаки:
-- признаки совеместимости (локация, медкнижка)  
-- признаки популярность смены/работодателя, временные признаки, признаки capacity  
-- исторические признаки, но без привязки к анализируйемой паре  
-- preference-признаки,т.е признаки предпочтения юзера и смены, введен признак warm-user по давности активности    
-- циклические признаки для часа и дня недели, для лучшего улавливания переодичности  
-- признаки as-of (на момент начала смены), избегает утечки, но накапливает историю  
-- recency-признаки (признаки давности взаимодействия смены и пользователя)  
-- признаки для новых пользователей (cold-start признаки)  

**Целевая переменная**
Расчитывается как 1 - Apply и Finished без Canceled смен, 0 - все остальное


**Обучение модели**
Пайплайн строиться на модели LightGBM, который хорошо справляется с нелинейными зависимостями и большим числом признаков. Гиперпараметры были подобраны через Randomized search, но с учетом потенциального переобучения модели (т.е. параметры для выбора были изначально простыми)
Для повышения стабильности предсказаний и снижения дисперсии использован ансамбль из трёх моделей с разными seed (42, 123, 456). Итоговая вероятность усредняется по трём моделям.
Для каждого seed создаётся pipeline, включающий стандартизацию числовых признаков, one‑hot‑кодирование категориальных (task_type) и сам классификатор LightGBM. Все три модели обучаются на одинаковых наборах X_train, y_train
Обучение сохраняет артефакты и модель. Результат обучения приложен в папке artifacts


**Целевая метрика**
Целевая метрика - constrained AUC (ROC‑AUC с ограничением FPR = min(1.0, capacity / 10)), расчитанная на тествоой выборке. Выведено в отдельную функцию.
Вероятности по ансамблю усредняются, затем рассчитывается регламентная метрика  и обычный ROC‑AUC.


**Проделанная работа**
За время работы над проектом были исследованы варианты:
- EDA по признакам и различные комбинации исторических признаков, например, парные взаимодействия, preference-признаки по историческим параметрам.
- построены корреляции между признаками
- были опробаваны логистическая регрессия с подбором параметров
- сделано несколько вариантов разбиения признаков на тренировочную и тестовую выборки.
- для снижения времени подсчета и построени признаков был сделан пробный вариант разделения даатсета на батчи и расчета признаков с сохраненеим по отдельно.
- был сделан альтернативный вариант для предсказания: вместо модели МЛ с вероятностями использован эврестический расчет весов.
Альернативные варианты не показали хороший результат на целевой метрике (в среднем показывали 0.5-0.64), в связи с чем был выбран финальный результат .
